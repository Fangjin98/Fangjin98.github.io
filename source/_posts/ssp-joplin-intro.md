*原文首次发表于[少数派](https://sspai.com/post/77020)(2022.12.05)。*

作为一个研究生，每天的工作就是大量的记录和思考，因此一个好的笔记应用对我至关重要。在先后使用过OneNote、VSCode和Notion作为自己的笔记应用后，我最终扎根在Joplin。

然而，前段时间看到并尝试申请Craft的Back to School计划，结果顺利通过了。在收到邮件后，我抱着不用白不用的心态注册了一个账号并将笔记从Joplin迁移到Craft。

在试用了半个月之后，我发现还是无法离开Joplin，以下是我的体验。

### 笔记应用在我生活中的定位

我的日常工作学习均深度与笔记应用绑定，不过笔记应用更主要在我的工作场景发挥作用。

在我看来，一个科研项目可以分为调研，生产以及展示这三个阶段。

1. 调研阶段：一个项目的开始首先需要确定明确的研究问题，避免做的问题是没有意义的或者别人已经解决了
2. 生产阶段：确定了研究问题，需要思考解决方案，例如算法或者系统设计；分析算法性能；最终进行实验查看实验效果
3. 展示阶段：科研工作的展示往往比科研工作本身还要重要。通常一项工作的展示形式是Word文档、论文、PPT以及博客文章等。

在上述三个阶段中，调研和生产经常会交替进行，且调研与生产也需要及时展示。因此，这三个阶段对笔记应用的需求有所不同。

在调研阶段，我需要**大量输入**，整理来自各种渠道的信息。由于很多内容是在开会交流时输入的，我希望它能够*秒开*。

在生产阶段，我会**不断试错**，总结经验，最终找到一个合适且有效的方案。在这一阶段，笔记应用是我的工作核心，我希望它

- *占用资源低*，这样可以常驻后台
- *支持[双链笔记](https://sspai.com/post/65273)*，更容易表达我的思考联系
- *支持笔记多开*，快速的在不同笔记间跳转

在展示阶段，我需要**时刻回顾**，并且针对不同的展示对象，将笔记内容转换为相应的形式，因此我希望它

- *跨平台支持完善*
- *多端同步方便快捷*，这一点与上一点相辅相成，只有同时做到这两点才能有很好的混合办公体验
- *支持Markdown记录*，即使通过不同软件展示，笔记的内容结构依旧不变

不过其中的很多应用设计目标也不仅仅是一个笔记应用，因此这些比较也许算是“以己之长，攻敌之短”。
接下来的内容我会重点比较Joplin和Craft的使用体验。事实上，我更喜欢使用Craft，但是Android客户端缺失让我实在无法正常使用（这也让我纠结是否需要将平板换成Ipad）。因此本文主要内容是将Joplin和Craft进行对比，说明为什么Joplin无法被替代。

### 除了Joplin一个能打的都没有吗

**Joplin谈不上完美，但是现阶段唯一能够满足我全部需求的应用。**

#### 外观

软件拥有优美的外观能吸引我使用，拥有过于优美的外观则会干扰我的注意力。

毫无疑问，不管是界面外观还是动画特效，Craft都远优于Joplin。

Craft所见即所得的文档，在编写过程中就可以看到文档最终的样式。并且Craft还提供了一些特殊效果，例如文字成块，渐变强调，中英文自动添加空格等，大大增强了文档的可玩性和美观上限。然而，对于一个笔记应用来说，我真的需要这些功能吗？

我很喜欢《Unix编程哲学》中的「文字是唯一接口」。所见即所得提供了笔记最终的样式，适用于观看，但是也容易干扰我的注意力。在换到Craft之后，我有很大一部分时间用在调整内容格式，而难以长时间专注于笔记内容本身。因此，我更喜欢直接与文本打交道。

Joplin在这方面提供了很高的自由度。它既提供了所见即所得的编辑器，也提供了双栏布局（左边编辑器，右边显示最终样式），还允许我使用外部编辑器编辑笔记。我使用最多的是双栏布局，其次是使用外部编辑器（例如 VSCode）编辑笔记。同时，Joplin提供了自定义CSS样式功能，因此我可以根据自己的喜好设计软件界面，或者使用[社区分享的主题](https://github.com/AngelCrawford/joplin-theme)。我还通过[Rich Markdown](https://github.com/CalebJohn/joplin-rich-markdown#readme)插件实现预览网页或其他笔记的功能，大大提高了查看器的实用性。

#### 启动速度

软件只有秒启动才能保证我不落下瞬间的信息。

Craft在Mac OS平台能够做到点开即用，但是在Windows平台有一些区别对待。Craft在Windows平台虽然能够做到点击即启动，但是从启动到笔记载入完成还需要一段时间，并且这段时间经常会花费许久。

![2023-05-05T215913](2023-05-05T215913.png)

我已经好几次碰到需要记录的时候点开软件迟迟无法输入，最终只能先记录在微信聊天框中。更烦人的是Windows版Craft会遇到笔记写作中断的情况。要么是在顶栏提示目前似乎是离线状态，要么直接整个界面不可用。这种小问题直接消磨我的使用欲望。

![2023-05-05T215921](2023-05-05T215921.png)

Joplin冷启动所需的时间相对较长，但是也在接受范围以内。启动完毕后，Joplin默认在后台常驻，因此关闭窗口后重新打开应用的反应速度很快，并且打开即可编辑笔记内容，在关键时刻从没掉过链子。

#### 内存占用

Windwos平台上的Craft功能相较于Mac OS平台比较简单，因此内存占用比较低。这是一个优点，但是考虑到它时不时崩溃的额外特性，我宁愿它多占用我一些内存，对电脑负担高总比对我负担高要好。

Joplin常驻后台的代价就是对内存消耗很大。在Windows平台上，关闭Joplin窗口仅保留后台服务就会消耗100多MB内存，内存消耗和Craft相当。打开Joplin就能达到300MB，虽然不算离谱，但是我还是希望笔记应用尽可能的小而美。

#### 双链功能

Craft支持双连@将内容链接到其他笔记中，并且链接的粒度很细，可以达到笔记中的每一行（块）。我很喜欢这个功能，因为这个功能可以很轻松地在笔记间建立联系，有助于我在回顾笔记时重建当时的思路。

Joplin原生不支持双链功能，得益于社区支持，通过[Note Link System](https://github.com/ylc395/joplin-plugin-note-link-system)插件实现双链笔记功能。该插件虽然不可以直接链接至文本行，但是可以链接到标题，也能够满足我的需求。如果真的想更细粒度的链接，可以手动为元素指定一个id，通过引用该id也可以实现双向链接。

不管对于内部笔记还是外部网页，Joplin采用相同的格式，即\[内容\](网站链接或内部笔记链接)。这样做的好处是，即使笔记导出至其他软件，笔记间的关系依然不会丢失。我在将Joplin的笔记迁移到Craft的时候，就发现双链完美的复现了。但是在将Craft的笔记重新导入回Joplin时，其双链是[[笔记标题]]的格式，Joplin默认无法解析这些双链。

我个人更喜欢Joplin的链接功能，对外部网页和内部笔记统一的格式，减少了学习成本并且迁移到其他笔记也不会碰到问题。

#### 笔记多开

没用过多标签页笔记功能之前，我以为自己不需要这个功能。用上了Craft的多标签页功能，才意识到这个功能有多香。

在日常生活中，有很多时候需要同时打开多个笔记。例如我会为每一个项目建立一个笔记用于汇总相关资料，每天的工作会记录在日记里。每天花一些时间将日记中的内容整理归纳到对应项目中。这个时候多开几个笔记页就可以方便的互相切换。

可惜的是，Windows版本的Craft缺失了我最喜欢的标签页功能，只提供了多窗口功能。虽然多窗口能够满足笔记迁移，但是无法让我重启应用后同时打开多个笔记。是的，多标签页还有一个作用就是能够保存当前的工作状态。我习惯关闭暂时不用的应用，以避免干扰。因此多开笔记使我在打开应用后第一时间知道我需要关注哪些笔记，更快地进入工作状态。

Joplin原生只支持一次开启一个笔记，但是万能的社区有[笔记多开插件](https://github.com/benji300/joplin-note-tabs)，安装后可以实现同时切换多个笔记的功能。这款插件的缺点可能就是美观程度不高，并且不能在多个设备同步多开的笔记。

#### 跨平台支持与多端同步

跨平台支持和多端同步是实现随时随地办公的重要特性，前者针对客户端，后者针对笔记内容，缺一不可。

对于大部分应用，例如 Craft ，这两个特性可以视作一个完整的功能。只要登陆 Craft 账号，就能在所有支持的平台同步笔记。这样绑定的缺点是，对于 Craft 不支持的平台 （Android 平台），我就无法查看编辑这些笔记。还好 Craft 开发了网页端，可以用来将就一下？**不幸的是，Craft 网页端不支持手机或平板等移动设备使用**。这就让我很费解，开发网页端的初衷不就是以较低成本借助浏览器实现全平台访问吗？结果非苹果手机平板无法在网页端查看笔记，又不提供对应的客户端，不就浪费了网页端的能力吗？

反观 Joplin 实现了客户端和笔记内容同步的解耦。首先 Joplin 提供了完善的客户端支（甚至[支持 Windows 3.1](https://discourse.joplinapp.org/t/joplin-for-windows-3-1/24799)），包括命令行应用，满足了以终端作为主要工作场景的用户。虽然我觉得在终端编辑软件的场景挺少的，但是后来想想，也许这是配合那些喜欢使用 Vim 编辑器的用户呢？还挺合理。

![2023-05-05T220039](2023-05-05T220039.png)

Joplin 还提供了多种数据同步方案，我可以使用付费的 Joplin Cloud 计划，也可以使用 Microsoft 账户同步，还可以使用 WebDAV 协议结合其他云盘服务同步。这些同步方案给予我足够的灵活性，让我可以自由地决定笔记数据放在哪里。如果觉得数据在云盘上不放心，Joplin 提供了笔记端到端加密功能，保证了笔记的安全性。不过我的工作内容目前来看没那么敏感，就是水水论文，因此没用过这个功能，就不额外介绍了。

#### Markdown支持

默认情况下，Craft 中复制的内容仅包含纯文本，也就是说一级标题、二级标题在粘贴后会全部变成普通文本，丢失笔记的层级内容。如果想要保留笔记层级信息，需要在侧边栏中选择复制 Markdown。

**这显然不够优雅。**

可能 Craft 的开发人员也觉得这种方式不优雅，直接在 Windows 端阉割了这一功能。当然，不仅仅是这一个功能被阉割，从下图可以看到，Craft Windows 端的动作相比较 macOS 明显减少，~~也让软件的优雅程度大大上升~~。

![2023-05-05T220118](2023-05-05T220118.png)

Joplin 使用左编辑右查看的双栏布局，因此我如果想复制 Markdown 文本，直接在编辑器中选中相应内容复制即可。如果想仅复制文本，在查看器中选中文本按下 Ctrl+C 即可，就这么简单。此外，可能是计算机系的职业病，对于一个系统，更希望知道其内部实现逻辑，也更习惯看代码。所以在笔记软件中，我更关注的是 Markdown 文本，而不是最终查看效果。因为笔记展示效果在每一个笔记软件中都不一样，但是 Markdown 文本是一致的。同时，Joplin 社区中也有一些插件扩展了原生的 Markdown 解析语法和编辑器等功能，增强软件的使用体验：

- [Enhancement](https://github.com/SeptemberHX/joplin-plugin-enhancement.git)：提供了解析Latex算法伪代码、渲染代码块、高级图片插入、高级表格编辑、解析:::语法等功能
- [Enhanced Editing](https://github.com/yangby/joplin-plugin-enhanced-editing)：增强了Joplin编辑器的体验
- [Rich Markdown](https://github.com/CalebJohn/joplin-rich-markdown#readme)：增强了Joplin查看器的体验，例如为链接前添加网站或笔记图标

### 总结：为什么 Joplin 好用

> 开闭原则：对修改封闭，对扩展开放——设计模式

我认为 Joplin 能够无感地融入工作，最重要的是对笔记原始内容做最少的修改，但是在笔记以外提供了大量插件从而增强我的使用体验。

在这一方面，Obsidian 做得很好，提供了高可玩性的插件系统。Joplin 虽然没有 Obsidian 那么多的插件，但是一个人能用到几个插件呢？相比较 Craft，Joplin 提供了一定的自由度，允许我们按需增加软件功能。相比较 Obsidian，Joplin 又避免太过自由，以至于软件学习曲线太陡峭。

Joplin 将自由程度控制在合适的范围，让我在使用过程中不至于总是想着把玩软件本身，也不至于碰到痛点手足无措，这可能就是我喜欢使用它的原因吧。
